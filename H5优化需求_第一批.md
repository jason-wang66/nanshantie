# 南山帖 H5 第一批优化需求

## 当前文件
`./白茶品牌/index.html`（单HTML文件，约2536行，CSS/JS全内联）

## 优化目标
提升转化链路：缩短入口路径 → 增强分享裂变 → 建立数据洞察 → 送礼场景转化

---

## 一、路径缩短 + 回访识别

### 1.1 名字输入改为可选
- 输入框 placeholder 改为"留下你的山客名（可不填）"
- "进入南山"按钮在输入框为空时也可点击
- 未填名字时，签文结果页用户名显示"无名山客"
- 填了名字时，正常显示

### 1.2 跳过按钮提前显示
- 开篇动画开始0.5秒后就显示"跳过 >"按钮（当前是动画开始就显示但视觉不够醒目）
- 按钮字号从12px提到14px，颜色从灰色改为 rgba(255,255,255,0.6)

### 1.3 回访用户识别
- 进入南山时，将用户名存入 localStorage（key: nanshantie_user）
- 再次打开H5时，检查 localStorage：
  - 有名字：跳过开篇动画，直接进入一木页，顶部显示"〇〇，今日茶运如何？"
  - 无名字但有记录：跳过开篇动画，直接进入一木页，顶部显示"山客，今日茶运如何？"
  - 首次访问：正常走开篇动画

## 三、数据埋点

### 3.1 埋点方案
使用最轻量的方案：封装一个 `track(event, data)` 函数，将事件发到指定端点。
暂时没有后端时，先 console.log 输出，并预留发送接口：
```javascript
function track(event, data = {}) {
    const payload = { event, timestamp: Date.now(), ...data };
    console.log('[track]', payload);
    // TODO: 替换为实际埋点服务
    // fetch('https://analytics.example.com/track', { method: 'POST', body: JSON.stringify(payload) });
}
```

### 3.2 五个关键事件

| 事件名 | 触发时机 | 附加数据 |
|--------|----------|----------|
| `intro_complete` | 开篇动画自然播放完毕 | { skipped: false } |
| `intro_skip` | 用户点击跳过 | { skip_time: 秒数 } |
| `guest_click` | 用户点击山客卡片 | { guest_id, guest_name, season } |
| `fortune_share` | 用户点击分享茶运 | { guest_id, guest_name, share_method } |
| `contact_click` | 用户点击结缘此茶 | { guest_id, guest_name, source: 'result'|'product' } |

## 四、送礼推荐——"以茶寄意"

### 4.1 入口位置
- 在底部导航栏增加第四个点"寄意"，或在一木页底部加"以茶寄意"入口按钮
- 推荐方案：在一木页山客网格下方，加一个横幅卡片："不知道选哪款？以茶寄意 →"

### 4.2 推荐弹窗（giftGuideModal）
点击后弹出一个全屏弹窗，包含：

**第一行：选择关系**
- 长辈 | 朋友 | 客户 | 恋人（四选一，胶囊按钮组）

**第二行：选择场景**
- 节日 | 感谢 | 问候 | 祝贺（四选一）

**推荐结果**（选择后立即显示，无需额外点击）：

- 每款显示：山客名 + 茶品名 + 一句推荐理由（30字内）+ 产品图
- 底部"结缘此茶"按钮
